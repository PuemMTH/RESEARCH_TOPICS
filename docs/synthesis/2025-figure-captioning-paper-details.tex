\section{Paper 1: LaMP-Cap}

\textbf{Full title:} LaMP-Cap: Personalized Figure Caption Generation With Multimodal Figure Profiles \\
\textbf{arXiv:} \href{https://arxiv.org/abs/2506.06561}{2506.06561} \quad
\textbf{Authors:} Ho Yin ``Sam'' Ng, Ting-Yao Hsu, Aashish Anantha Ramakrishnan และคณะ \\
\textbf{Repo files:} papers/8-LaMPCap\_2025.* และ docs/synthesis/8-LaMPCap\_2025-analysis.html

\subsection{Core Idea}
LaMP-Cap เปลี่ยนโจทย์จาก ``generate caption จาก target figure อย่างเดียว'' เป็น ``generate caption จาก target figure พร้อม profile figures จาก paper เดียวกัน'' โดยถือว่า profile figures สะท้อนทั้ง domain context, writing style, terminology และวิธีที่ผู้เขียนอธิบาย visual evidence ใน paper นั้น

\begin{note}
\textbf{แก่นของ paper:} ถ้าอยากให้ caption ไม่ generic ต้องให้โมเดลเห็นตัวอย่าง caption/figure อื่นจาก paper เดียวกัน เพราะ scientific captions มักมี style และ vocabulary เฉพาะของ paper/author
\end{note}

\subsection{Task Definition แบบละเอียด}
\begin{longtable}{@{}L{3cm}L{11.6cm}@{}}
\toprule
\textbf{Element} & \textbf{Description} \\
\midrule
Target figure & รูปที่ต้อง generate caption \\
Target source & target image + figure-mentioning paragraphs เช่นย่อหน้าที่มี ``Figure 3 shows...'' \\
Profile figures & รูปอื่นจาก paper เดียวกัน สูงสุด 3 รูป \\
Profile source & profile image + profile figure-mentioning paragraph \\
Profile caption & caption จริงของ profile figure ใช้เป็น signal ด้าน style/structure \\
Output & caption ของ target figure ที่ควรใกล้ author-written caption \\
\bottomrule
\end{longtable}

\subsection{Dataset Construction}
\begin{itemize}
\item Source dataset คือ SciCap Challenge Dataset จาก arXiv papers
\item เลือกหนึ่ง figure ต่อ paper เป็น target และใช้ figures ที่เหลือเป็น profile
\item paper ที่มี figure เดียวถูกตัดออก เพราะไม่มี profile ให้ใช้
\item จำกัด profile สูงสุด 3 รูป เพราะ SciCap Challenge จำกัดจำนวน figure ต่อ paper
\item train/validation/test ตาม split เดิม 80/10/10
\end{itemize}

\begin{note}
\textbf{Scale:} 110,828 target figures; 197,075 profile figures; 307,903 total figures. Figure types หลักคือ Graph Plot, Node Diagram, Equation, Bar Chart, Scatterplot.
\end{note}

\subsection{Experimental Setup}
\begin{longtable}{@{}L{3.2cm}L{11.4cm}@{}}
\toprule
\textbf{Setting} & \textbf{Meaning} \\
\midrule
No-Profile & ใช้ target image + paragraph เท่านั้น \\
One-Profile & เพิ่ม profile figure หนึ่งชุดจาก paper เดียวกัน \\
All-Profile & ใช้ profile figures ทั้งหมดที่มี \\
Same-paper profile & profile มาจาก paper เดียวกัน ใช้วัด personalization/context effect \\
Metric & BLEU, ROUGE, BERTScore และบางการทดลองมี human evaluation \\
\bottomrule
\end{longtable}

\subsection{Key Findings}
\begin{itemize}
\item Profile information ช่วยให้ generated caption ใกล้ ground-truth author captions มากขึ้นในหลาย model
\item All-Profile มักดีที่สุด แต่ gain หลักเกิดจาก No-Profile ไป One-Profile แปลว่า context ชิ้นแรกที่ relevant สำคัญมาก
\item ใน ablation, profile caption สำคัญที่สุด เพราะให้ทั้ง style และ target format; profile image สำคัญกว่า paragraph ในหลายกรณี แปลว่า multimodal profile มีประโยชน์จริง ไม่ใช่แค่ text context
\item BERTScore ระหว่าง target และ profile captions สูง แต่ ROUGE-L กระจายกว้าง แปลว่า profile captions มี semantic relatedness แต่ไม่ได้ copy wording เหมือนกัน
\item Qwen-2.5-VL-7B มี failure rate สูงกว่า model อื่นในบาง profile setting เป็น warning ว่า open-source VLM ต้องมี validation/fallback
\end{itemize}

\subsection{Limitations ที่ควรอ้างเป็น Gap}
\begin{itemize}
\item Context ยังเป็น \textbf{profile figures ที่ curated แล้ว} ไม่ใช่ context ที่ retrieve จาก PDF จริง
\item ยังไม่ตอบว่า context จาก abstract, methodology, results section, nearby text หรือ cross-reference ช่วยมากกว่า profile figures หรือไม่
\item การ optimize ให้ใกล้ author-written caption อาจไม่เท่ากับการได้ caption ที่ factual หรือ useful เพราะ original caption อาจสั้น/คลุมเครือ
\item Evaluation ยังพึ่ง lexical/semantic similarity metrics เป็นหลัก
\end{itemize}

\subsection{How to Use in Thesis}
\begin{itemize}
\item ใช้ LaMP-Cap เป็น baseline dataset สำหรับทดลอง context-aware captioning
\item เปลี่ยนจาก fixed profile context เป็น RAG ที่ retrieve context หลายระดับจาก paper
\item ทำ ablation เทียบ: target only, mention paragraph, OCR, section context, profile figure, retrieved full-document context
\item ใช้ finding ``One-Profile gives large gain'' เป็นเหตุผลว่า retrieval quality สำคัญกว่า context length
\end{itemize}

\section{Paper 2: Leveraging Author-Specific Context for Scientific Figure Caption Generation}

\textbf{Full title:} Leveraging Author-Specific Context for Scientific Figure Caption Generation: 3rd SciCap Challenge \\
\textbf{arXiv:} \href{https://arxiv.org/abs/2510.07993}{2510.07993} \quad
\textbf{Authors:} Watcharapong Timklaypachara, Monrada Chiewhawan, Nopporn Lekuthai, Titipat Achakulvisut \\
\textbf{Repo files:} papers/9-AuthorContextSciCap\_2025.*

\subsection{Core Idea}
Paper นี้เป็น system paper สำหรับ SciCap Challenge ที่ใช้ LaMP-Cap แล้วออกแบบ two-stage pipeline: ขั้นแรกสร้าง caption ที่ content-grounded ด้วย context filtering + category-focused prompt optimization ขั้นที่สอง refine caption ด้วย profile figures เพื่อให้ style ใกล้ paper ต้นฉบับ

\begin{note}
\textbf{แก่นของ paper:} ถ้าเอา paragraph ส่งเข้า LLM ตรง ๆ จะเกิด noisy caption และ hallucination จึงต้อง filter context ก่อน แล้วค่อย refine style ด้วย profile examples
\end{note}

\subsection{Pipeline แบบละเอียด}
\begin{longtable}{@{}L{3cm}L{5.7cm}L{5.8cm}@{}}
\toprule
\textbf{Module} & \textbf{Input/Process} & \textbf{Purpose} \\
\midrule
Context filtering & paragraph data จาก target figure & ลด irrelevant content ก่อน generation \\
Category-focused prompt & paper category เช่น CS, Math, Physics, Statistics & ให้ prompt ใช้ terminology/structure ที่เหมาะกับ domain \\
MIPROv2 & bootstrap instruction-example pairs + Bayesian optimization & optimize prompt โดยเน้น ROUGE-L precision \\
SIMBA & feedback-driven optimization & หา reasoning path และแก้ problematic cases \\
Candidate selection & captions จากหลาย category/prompt & เลือก caption ที่ alignment ดีกว่า \\
Profile refinement & profile figures/captions เป็น few-shot examples & ปรับ style, length, conciseness และ author alignment \\
\bottomrule
\end{longtable}

\subsection{Important Design Choices}
\begin{itemize}
\item ใช้ category-focused prompts เพราะ scientific papers ต่าง field มี caption convention ต่างกัน
\item paper ที่มีหลาย category สร้างหลาย candidate แล้วเลือกภายหลัง แทนการบังคับให้ prompt เดียวครอบคลุมทุก field
\item Stage 2 กำหนดความยาวประมาณ \(\pm 15\%\) เพื่อแก้ปัญหา caption ที่ verbose จาก Stage 1
\item ใช้ profile captions เป็น structural reference ไม่ใช่แค่ข้อมูล factual
\item ใช้ Gemini-2.5-flash เป็น reranker เพื่อเน้น precision/style consistency
\end{itemize}

\subsection{Results แบบละเอียด}
\begin{longtable}{@{}L{4.1cm}L{4.3cm}L{6.2cm}@{}}
\toprule
\textbf{Experiment} & \textbf{Reported result} & \textbf{Interpretation for thesis} \\
\midrule
Filtered paragraph + prompt optimization & ROUGE-1 recall 0.4867 \(\rightarrow\) 0.5443; ROUGE-2 recall 0.2223 \(\rightarrow\) 0.2572 & Filtering ช่วย coverage แต่ต้องระวัง precision ลด \\
Category-focused prompting & ROUGE-1 recall +8.3\%; BLEU-4 loss จำกัดกว่า non-category prompt & Domain-aware generation สำคัญสำหรับ academic caption \\
Non-category-focused prompting & recall เพิ่มมาก แต่ precision ลดหนัก & context เยอะ/กว้างเกินไปทำให้ caption verbose และ off-topic \\
Gemini reranking & ROUGE precision +3.46 ถึง +4.43\% & มี post-generation selector ช่วยลด caption ที่ไม่ตรงประเด็น \\
Profile-informed refinement & BLEU +40--48\%; ROUGE precision +25--27\% & profile context ช่วย style และ conciseness ชัดเจน \\
\bottomrule
\end{longtable}

\subsection{Limitations ที่ควรอ้างเป็น Gap}
\begin{itemize}
\item ยังเป็น prompt-engineering-heavy pipeline ต้องใช้ DSPy optimizer, reranker และ few-shot refinement หลายชั้น
\item Category-focused prompt ไม่ใช่ retrieval จาก document structure จริง เช่น section, layout, table, figure proximity
\item ใช้ BLEU/ROUGE เป็นหลัก ทำให้วัด lexical overlap มากกว่า visual faithfulness
\item ไม่ได้เสนอ full PDF ingestion pipeline จึงยังไม่แก้โจทย์ end-to-end academic document understanding
\end{itemize}

\subsection{How to Use in Thesis}
\begin{itemize}
\item ใช้เป็น argument ว่า \textbf{context filtering is necessary}
\item เปลี่ยน context filtering จาก prompt/manual rule เป็น retrieval module: mention detector, section-aware retrieval, semantic retrieval, OCR alignment
\item เพิ่ม evaluator ที่ตรวจ hallucination หลัง generation แทนพึ่ง reranker จาก metric overlap อย่างเดียว
\item ทำ comparison: prompt-optimized context vs RAG-retrieved context
\end{itemize}

\section{Paper 3: Personalized Scientific Figure Caption Generation}

\textbf{Full title:} Personalized Scientific Figure Caption Generation: An Empirical Study on Author-Specific Writing Style Transfer \\
\textbf{arXiv:} \href{https://arxiv.org/abs/2509.25817}{2509.25817} \quad
\textbf{Authors:} Jaeyoung Kim, Jongho Lee, Hongjun Choi, Sion Jang \\
\textbf{Repo files:} papers/10-PersonalizedSciFigCap\_2025.*

\subsection{Core Idea}
Paper นี้ถามตรง ๆ ว่า author profile data ช่วย personalized figure captioning แค่ไหน และที่สำคัญกว่านั้นคือการทำให้ caption เหมือน author-written caption อาจไม่เท่ากับการทำให้ caption คุณภาพดีขึ้น ผู้เขียนจึงเพิ่ม quality evaluator และทดลอง quality-aware generation

\begin{note}
\textbf{แก่นของ paper:} Context/profile เพิ่ม BLEU/ROUGE ได้ แต่ metric เหล่านี้อาจ reward style imitation มากกว่า informativeness จึงต้องแยก quality objective ออกจาก style objective
\end{note}

\subsection{Architecture และ Input}
\begin{longtable}{@{}L{3.2cm}L{11.4cm}@{}}
\toprule
\textbf{Component} & \textbf{Detail} \\
\midrule
\(f_{quality}\) & Qwen-2.5-VL-3B fine-tuned เพื่อให้คะแนน caption quality 1--6 \\
Quality labels & สร้าง synthetic labels ด้วย GPT-4.1 จาก figure-caption pairs \\
\(g_{caption}\) & Qwen-2.5-VL-7B fine-tuned เป็น caption generator \\
Target inputs & figure image \(F\), explanatory paragraphs \(P\), textual mentions \(M\), OCR text \(O\) \\
Profile inputs & related figures จาก paper เดียวกัน พร้อม \(F, P, M, O, C\) \\
Training filter & ใช้เฉพาะ samples ที่ \(f_{quality}\) ทำนาย quality score \(\geq 3\) \\
\bottomrule
\end{longtable}

\subsection{Quality Evaluator Details}
\begin{itemize}
\item สร้าง synthetic quality assessment dataset 3,000 samples ด้วย GPT-4.1
\item fine-tune Qwen-2.5-VL-3B เพื่อประเมิน score 1 = low quality ถึง 6 = high quality
\item validation 200 samples ได้ Spearman rank correlation 0.759 และ Quadratic Weighted Kappa 0.754
\item เหตุผลที่ต้องมี evaluator: caption จาก arXiv จำนวนมากมีคุณภาพต่ำ การ train ตรง ๆ จาก reference อาจทำให้ model copy low-quality captions
\end{itemize}

\subsection{Caption Generator Details}
\begin{itemize}
\item ใช้ Qwen-2.5-VL-7B เป็น generator เพราะเป็น open-source VLM ที่ fine-tune ได้
\item fine-tune 1 epoch ด้วย AdamW, learning rate \(2e{-5}\), batch size 2
\item input รวม target context และ profile context จาก paper เดียวกัน
\item ประเมินบน personalized SciCap/LaMP-Cap test set ด้วย BLEU และ ROUGE
\end{itemize}

\subsection{Results แบบละเอียด}
\begin{longtable}{@{}L{4.2cm}L{4.2cm}L{6.2cm}@{}}
\toprule
\textbf{Finding} & \textbf{Number} & \textbf{Interpretation} \\
\midrule
Fine-tuned Qwen ชนะ Gemini บน full test & BLEU-4 0.304 vs 0.278; ROUGE-L 0.555 vs 0.540 & open-source VLM fine-tuning เป็น baseline ที่ทำได้จริงสำหรับ thesis \\
เพิ่ม paragraph จาก F+C baseline & BLEU-4 0.123 \(\rightarrow\) 0.295; ROUGE-L 0.355 \(\rightarrow\) 0.545 & paragraph context เป็น signal ใหญ่มาก ไม่ควรทำ image-only \\
เพิ่ม mention และ OCR & full config BLEU-4 0.304; ROUGE-L 0.555 & mention/OCR เพิ่ม gain แบบ incremental แต่สำคัญสำหรับ factual detail \\
profile quantity effect & 3 profile figures มักได้คะแนนสูงสุด & context มากขึ้นช่วย แต่ต้องดูคุณภาพ context ด้วย \\
Generated quality issue & Gemini และ Qwen ได้ quality score เฉลี่ยราว 3 และต่ำกว่า author captions & style-matching ไม่พอ ต้องมี quality objective \\
Forced-Q6 & quality score สูงขึ้น แต่ ROUGE-L อาจลด & metric overlap อาจลงโทษ caption ที่ informative กว่า reference \\
\bottomrule
\end{longtable}

\subsection{Quality-Style Trade-off}
ประเด็นที่ควรเอาไปเขียน research gap คือ style fidelity และ caption quality อาจอยู่คนละ objective โมเดลที่ optimize ให้ใกล้ author caption อาจเรียนรู้ wording/style ของผู้เขียน แต่ไม่ได้ guarantee ว่า caption factual, informative หรือ useful กว่าเดิม ในกรณี reference caption คุณภาพต่ำ การ generate caption ที่ดีกว่าอาจทำให้ ROUGE ลด เพราะ wording ไม่ตรง reference

\subsection{Limitations ที่ควรอ้างเป็น Gap}
\begin{itemize}
\item Quality evaluator พึ่ง GPT-4.1 labels จึงยังมี judge bias และควรตรวจด้วย human/domain expert
\item Metrics หลักยังเป็น BLEU/ROUGE ทำให้แยกไม่ได้ว่า score สูงเพราะ factual ดีหรือแค่ wording เหมือน
\item ใช้ LaMP-Cap fields ที่ curated แล้ว ยังไม่ได้ parse PDF เอง
\item ยังไม่ประเมิน visual faithfulness และ context faithfulness อย่างเป็นระบบ
\end{itemize}

\subsection{How to Use in Thesis}
\begin{itemize}
\item ใช้ Qwen-2.5-VL-7B fine-tuned เป็น open-source baseline
\item ใช้ input components \(F, P, M, O\) เป็น ablation design สำหรับ thesis
\item เพิ่ม evaluation dimensions: visual faithfulness, context relevance, academic informativeness, style similarity
\item ใช้ quality-aware idea เป็น refinement/checking module หลัง generation
\end{itemize}

\section{Research Gap ที่เขียนต่อใน Proposal ได้ทันที}

\textbf{Working thesis claim:} Existing 2025 work shows that context improves scientific figure captioning, but current methods rely on curated profile/context fields and lexical metrics. This thesis will investigate an end-to-end, layout-aware, retrieval-based pipeline that extracts context directly from academic PDFs and evaluates generated captions for visual faithfulness, context relevance, and academic informativeness.
