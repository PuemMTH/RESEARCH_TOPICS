SciCap: Generating Captions for Scientific Figures

EMNLP 2021

🎯 Overview

SciCap (Hsu et al., 2021) เป็นงานวิจัยพื้นฐานที่สร้าง Large-scale Dataset สำหรับ Scientific Figure Captioning จากบทความวิชาการใน arXiv สาขา Computer Science กว่า 10 ปี พร้อมทั้งเสนอ Neural Baseline Framework สำหรับการสร้างคำอธิบายรูปกราฟอัตโนมัติ

📌 จุดประสงค์หลักของ Paper

ปัญหาที่ Paper นี้แก้
  • ก่อน SciCap ไม่มี benchmark dataset ขนาดใหญ่ สำหรับ scientific figure captioning — งานก่อนหน้าใช้ dataset เล็กหรือ domain เฉพาะ ทำให้เปรียบเทียบ model ได้ยาก
  • ไม่มี baseline ที่ชัดเจน ว่า neural approach ทำได้ดีแค่ไหนบน real-world scientific figures
  • ไม่มีการวิเคราะห์ ว่า caption ของ scientific figures มีลักษณะอย่างไร (ยาวแค่ไหน, มีกี่ประโยค, ภาษาเป็นอย่างไร)
  • เป้าหมายที่ 1: สร้าง large-scale, publicly available dataset จาก arXiv เพื่อเป็น shared benchmark
  • เป้าหมายที่ 2: วิเคราะห์ลักษณะของ caption ในเอกสารวิชาการจริง (caption statistics, first-sentence patterns)
  • เป้าหมายที่ 3: สร้าง neural baseline เพื่อวัดว่า figure-only approach ทำได้ดีแค่ไหน และเผยจุดอ่อนของ approach นี้
  • เป้าหมายที่ 4: วางมาตรฐาน evaluation metrics (BLEU-4, METEOR, ROUGE-L, CIDEr) สำหรับ task นี้

📊 Dataset Construction

  • แหล่งข้อมูล: arXiv CS papers (2010-2020) ~290K papers
  • เครื่องมือ: PDFFigures 2.0 สำหรับดึง figure-caption pairs
  • ขนาด: ~416K figure-caption pairs (หลังกรอง: Graph plots เท่านั้น)
  • การกรอง: เก็บเฉพาะ graph plots (line, bar, scatter) ตัด diagrams/photos ออก
  • แบ่งข้อมูล: Train/Val/Test ตาม figure-level split

🔬 Methodology — Neural Baseline

Pipeline Architecture Input: Figure Image → CNN Encoder (ResNet-101) → Visual Features → LSTM Decoder → Generated Caption + Preprocessing: Text normalization, subfigure filtering + Training: Cross-entropy loss → SCST (Self-Critical Sequence Training) + Evaluation: BLEU-4, METEOR, ROUGE-L, CIDEr

📈 Key Results

  • Caption ส่วนใหญ่มี 1 ประโยค (~60%) → เป็น first-sentence prediction task
  • Text normalization ไม่ช่วยเพิ่มประสิทธิภาพ
  • SCST ช่วยเพิ่ม CIDEr score อย่างมีนัยสำคัญเทียบกับ Cross-Entropy
  • คะแนนโดยรวมยังต่ำ → แสดงว่า Figure-only approach มีข้อจำกัด ต้องการ context เพิ่มเติม
🔗 ความเกี่ยวข้องกับวิทยานิพนธ์ (เจาะลึก)
  • Dataset: SciCap เป็น standard benchmark ที่งานเราจะใช้ evaluate — ทำให้เปรียบเทียบกับงานอื่นได้โดยตรง
  • Baseline สำหรับ ablation: Figure-only (ResNet + LSTM) เป็น baseline ระดับล่างสุดในการทดลองของเรา เพื่อพิสูจน์ว่า RAG + VLM ทำได้ดีกว่า
  • Research gap: ผลลัพธ์ที่ต่ำของ figure-only baseline คือหลักฐานที่เราอ้างได้ว่า "ทำไมต้องใช้ document context" ซึ่งตรงกับ วัตถุประสงค์ข้อ 2 ของวิทยานิพนธ์เรา (พัฒนากระบวนการวิเคราะห์รูปภาพร่วมกับบริบท)
  • Task definition: เรา reuse การ define task (first-sentence prediction, graph plots focus) จาก SciCap เป็น scope เริ่มต้น
  • Metrics: BLEU-4, METEOR, CIDEr จาก SciCap เป็น automatic metrics หลักที่เราต้องรายงาน

MLBCAP: Multi-LLM Collaborative Figure Caption Generation

AAAI 2025 Workshop

🎯 Overview

MLBCAP เป็น State-of-the-art framework ที่ใช้ Multi-LLM Collaborative approach สำหรับสร้าง caption ที่มีคุณภาพสูง โดยใช้ LLM หลายตัวทำงานร่วมกันในแต่ละขั้นตอน: Data Cleaning, Caption Generation, Quality Judgment, และ Post-Editing

📌 จุดประสงค์หลักของ Paper

ปัญหาที่ Paper นี้แก้
  • Training data quality: SciCap dataset มี caption คุณภาพต่ำจำนวนมาก (ผู้เขียน paper บางคนเขียน caption ไม่ดี) → ทำให้ model ที่ train บน dataset นี้ถูก "สอน" ด้วยตัวอย่างที่ไม่ดี
  • Single model bottleneck: การใช้ VLM ตัวเดียวทำทุกอย่าง (perception + reasoning + generation + quality check) ทำให้ optimize ได้ยาก
  • Lack of quality control: ไม่มีกลไกตรวจสอบว่า caption ที่สร้างขึ้นมีคุณภาพพอก่อน output
  • เป้าหมายที่ 1: สร้าง data cleaning pipeline ที่ใช้ LLM กรอง caption คุณภาพต่ำออกจาก SciCap → ได้ SciCap-refined dataset
  • เป้าหมายที่ 2: แบ่ง caption generation task ออกเป็น subtasks → assign LLM ที่เหมาะสมให้แต่ละ role
  • เป้าหมายที่ 3: เพิ่ม quality judgment loop เพื่อเลือก caption ที่ดีที่สุดก่อน output
  • เป้าหมายที่ 4: บรรลุ SOTA บน SciCap benchmark ทุก metrics

🔬 4-Stage Framework

MLBCAP Pipeline Stage 1: Quality Assessment (LLM₁) → ประเมินคุณภาพ caption ดั้งเดิม → คัดกรอง low-quality ออก Stage 2: Caption Generation (LLM₂ = VLM) → สร้าง candidate captions จาก figure + context → ใช้ VLM เช่น GPT-4V, LLaVA Stage 3: Quality Judgment (LLM₃) → เปรียบเทียบ original vs generated caption → ให้คะแนนและเลือก caption ที่ดีกว่า Stage 4: Post-Editing (LLM₄) → ปรับปรุง caption ที่เลือก → เพิ่มความถูกต้อง/ครบถ้วน

📈 Key Contributions

  • Multi-LLM Collaboration: แต่ละ LLM มีบทบาทเฉพาะ ไม่ใช่ใช้ model เดียวทำทุกอย่าง
  • Quality-Aware Pipeline: กรองข้อมูลคุณภาพต่ำก่อนนำไป train
  • SciCap-refined Dataset: Dataset ใหม่ที่ผ่านการกรองและปรับปรุงคุณภาพ
  • ผลลัพธ์: SOTA บน SciCap benchmark ทุก metrics (BLEU, METEOR, CIDEr)
🔗 ความเกี่ยวข้องกับวิทยานิพนธ์ (เจาะลึก)
  • Pipeline structure ที่เราต่อยอด: เรายืม 4-stage logic มา แต่แทน proprietary LLM (GPT-4V) ด้วย open-source VLM (InternVL/LLaVA) + เพิ่ม RAG เป็น Stage 1.5 ระหว่าง figure input กับ generation
  • Modular design = ablation ง่าย: การแยก stage ชัดเจนทำให้เราสามารถทดสอบว่า "RAG context ช่วยที่ stage ไหนมากที่สุด" → ตอบ วัตถุประสงค์ข้อ 4 (ประเมินคุณภาพ)
  • Research gap ที่เราเติม: MLBCAP ไม่ได้ใช้ RAG — context ที่ใส่ไปเป็น fixed mention-paragraph แบบ SciCap+ งานเราเพิ่ม semantic retrieval ทำให้ context ที่ได้มีความเกี่ยวข้องมากขึ้น
  • Open-source gap: MLBCAP พึ่ง GPT-4V ซึ่งมีค่าใช้จ่ายและไม่ reproducible → งานเราใช้ open-source ทั้งหมด ซึ่งเป็น contribution เพิ่มเติม
  • Quality judgment เป็น evaluation framework: แนวคิด LLM-as-judge ใน Stage 3 นำมาใช้เป็นส่วนหนึ่งของ evaluation ของเรา ควบคู่กับ BLEU/CIDEr

DePlot: One-shot Visual Language Reasoning by Plot-to-Table Translation

ACL 2023 · Google

🎯 Overview

DePlot (Liu et al., 2023 — Google Research) เสนอแนวคิดแยก visual encoding ออกจาก reasoning โดยแปลง chart/plot เป็น linearized data table ก่อน แล้วส่งให้ LLM ทำ reasoning ต่อ ทำให้ใช้ few-shot/one-shot prompting ได้โดยไม่ต้อง fine-tune

📌 จุดประสงค์หลักของ Paper

ปัญหาที่ Paper นี้แก้
  • End-to-end models ผสม perception กับ reasoning: เมื่อ model ทำ chart QA แบบ end-to-end มันต้อง learn ทั้งการ "อ่าน" chart และการ "คิด" วิเคราะห์พร้อมกัน → ยากที่จะปรับปรุงแต่ละส่วน
  • Fine-tuning ต้องการ data มาก: การ fine-tune LLM สำหรับแต่ละ chart domain ต้องการ labeled data จำนวนมาก ซึ่งหายาก
  • Chart perception ไม่แม่นยำ: LLM ที่รับ image โดยตรงมักอ่านค่าตัวเลขหรือ label ผิดพลาด (hallucinate)
  • เป้าหมายที่ 1: แยก "chart perception" ออกจาก "numerical reasoning" เป็น 2 modules ที่ upgrade แยกกันได้
  • เป้าหมายที่ 2: พิสูจน์ว่า chart → text table เป็น intermediate representation ที่ดี สำหรับให้ LLM ทำ reasoning ต่อโดยไม่ต้อง fine-tune
  • เป้าหมายที่ 3: สร้าง metric ใหม่ (RMS) สำหรับวัดความแม่นยำของ chart-to-table conversion
  • เป้าหมายที่ 4: แสดงว่า one-shot prompting + DePlot ทำได้ดีกว่า fully fine-tuned SOTA ก่อนหน้า

🔬 Methodology

DePlot Architecture Step 1: Plot-to-Table (Visual Encoder) Chart Image → Pix2Struct (fine-tuned) → Linearized Text Table "Title | x_label | y_label \n val1 | val2 | val3 ..." Step 2: Table-to-Answer (LLM Reasoning) Linearized Table + Question → LLM (PaLM/Codex) → Answer ใช้ few-shot prompting, ไม่ต้อง fine-tune LLM

📊 RMS Metric (Relative Mapping Similarity)

DePlot เสนอ metric ใหม่ RMS สำหรับวัดคุณภาพการแปลง chart → table ซึ่งเน้นว่าค่าสัมพัทธ์ (relative magnitude) ถูกต้อง แม้ค่าตัวเลขจะไม่ตรง exact match

📈 Key Results

  • One-shot DePlot + LLM ทำได้ใกล้เคียงหรือดีกว่า SOTA ที่ fine-tune เต็มรูปแบบ
  • Chart-to-table accuracy สูงกว่า ChartOCR อย่างมาก
  • ใช้ได้กับหลาย chart types: bar, line, pie, scatter
  • แยก perception กับ reasoning ทำให้อัปเกรดแต่ละส่วนได้อิสระ
🔗 ความเกี่ยวข้องกับวิทยานิพนธ์ (เจาะลึก)
  • หลักการ "perception-reasoning separation": นี่คือ design principle สำคัญของ pipeline เรา — ใช้ VLM สำหรับ extract ข้อมูลจาก figure (perception) แยกออกจาก LLM ที่ generate caption (reasoning) ป้องกัน hallucination
  • Intermediate representation สำหรับ charts: ในงานเรา เมื่อ figure เป็น graph/plot เราสามารถใช้ chart→table เป็นขั้นกลางก่อนส่งให้ LLM generate caption → ลด perceptual error ตาม pattern ของ DePlot
  • RMS metric: สำหรับ evaluation ในกรณีที่ caption พูดถึงค่าตัวเลข เราอาจนำ RMS มาเสริม BLEU/CIDEr เพื่อวัดความแม่นยำของตัวเลขที่พูดถึง
  • ปัญหาที่ DePlot ยังไม่แก้: DePlot ไม่ใช้ document context เลย — รู้แค่ข้อมูลในกราฟ แต่ไม่รู้ว่ากราฟนี้อยู่ใน paper เรื่องอะไร → นี่คือช่องว่างที่งานเราเติม ด้วย RAG
  • ตอบ วัตถุประสงค์ข้อ 2: แนวคิดการแปลง figure เป็น structured representation ก่อน เป็นส่วนหนึ่งของ "กระบวนการวิเคราะห์รูปภาพ" ในวิทยานิพนธ์เรา

SciCap+: A Knowledge-Augmented Dataset for Scientific Figure Captioning

AAAI 2023

🎯 Overview

SciCap+ (Yang et al., 2023) ขยาย SciCap dataset โดยเพิ่ม mention-paragraphs (ย่อหน้าที่กล่าวถึงรูป) และ OCR tokens เข้าไป แล้ว reframe ปัญหาจาก "figure-to-caption" เป็น "knowledge-augmented image captioning"

📌 จุดประสงค์หลักของ Paper

ปัญหาที่ Paper นี้แก้
  • SciCap ใช้ figure เดียว: งานก่อนหน้า (SciCap baseline) พิสูจน์แล้วว่า figure-only ทำได้ไม่ดี แต่ยังไม่มีใครทดสอบอย่างเป็นระบบว่า "context แบบไหนช่วยได้มากที่สุด"
  • ขาด document-level knowledge: caption ที่ดีต้องรู้ว่ากราฟนี้บอกอะไร ซึ่งมักอยู่ใน body text ของ paper ไม่ใช่แค่ visual features
  • Data leakage ใน SciCap split: SciCap แบ่ง train/test แบบ figure-level ทำให้ figures จาก paper เดียวกันอาจอยู่ทั้ง train และ test → model อาจ "จำ" paper แทนที่จะ learn
  • เป้าหมายที่ 1: เพิ่ม mention-paragraphs และ OCR tokens เข้า dataset เพื่อสร้าง knowledge-augmented version
  • เป้าหมายที่ 2: ทดสอบ ablation อย่างเป็นระบบ — วัดว่า (figure only) vs (figure + OCR) vs (figure + mention) vs (mention + OCR ไม่มี figure) ให้ผลต่างกันแค่ไหน
  • เป้าหมายที่ 3: แก้ data leakage โดย re-split ที่ document-level เพื่อให้ผลการประเมินน่าเชื่อถือกว่าเดิม
  • เป้าหมายที่ 4: พิสูจน์ empirically ว่า document context (mention-paragraphs) มีผลต่อคุณภาพ caption มากกว่า visual features

📊 Dataset Augmentation

  • Mention-paragraphs: ดึงจาก PDF body text ด้วย PDFFigures 2.0 + regex matching
  • OCR tokens: ใช้ Google Vision OCR API ดึงข้อความ + bounding box coordinates
  • ขนาด: ~414K figures พร้อม augmented metadata
  • Re-split: แบ่งข้อมูลใหม่ที่ document-level (ไม่ใช่ figure-level) เพื่อป้องกัน data leakage

🔬 Model: M4C-Captioner

M4C-Captioner Architecture Input Modalities: 1. Figure → ResNet-152 → Visual Features (2048-d) 2. Mention-paragraph → SciBERT → Text Features (768-d, max 192 tokens) 3. OCR tokens → FastText + PHOC + Faster R-CNN → Multi-modal Features All features → M4C Transformer (4 layers, 12 heads) → Caption + Pointer Network: เลือกคำจาก vocabulary หรือ copy จาก OCR tokens

📈 Critical Finding

🔑 ผลลัพธ์สำคัญที่สุด
  • Mention-paragraph เพียงอย่างเดียว ให้ผลดีกว่า figure-only อย่างมาก
  • Mention + OCR (ไม่มี figure) ให้ผลดีที่สุด → visual feature จาก ResNet เป็น noise!
  • Model ชนะ Human ในทุก automatic metrics เมื่อมี context
  • Human evaluation: แม้มี mention-paragraph มนุษย์ยังเขียน caption ได้ไม่ดี → task นี้ยากมาก
🔗 ความเกี่ยวข้องกับวิทยานิพนธ์ (เจาะลึก)
  • หลักฐานตรงที่สุดสำหรับ hypothesis เรา: SciCap+ พิสูจน์ว่า mention-paragraph สำคัญกว่า visual features — นี่คือ empirical justification สำคัญที่สุดสำหรับ วัตถุประสงค์ข้อ 2 ของวิทยานิพนธ์เรา
  • ข้อจำกัดของ SciCap+ ที่เราแก้: mention-paragraph ใน SciCap+ ดึงด้วย regex matching อย่างง่าย → มักได้ context ที่ไม่ครบหรือ noisy — งานเราใช้ RAG (semantic similarity retrieval) เพื่อดึง context ที่เกี่ยวข้องได้ดีกว่า
  • Visual encoder ที่ดีกว่า: ResNet เป็น noise → งานเราใช้ modern VLM (InternVL, LLaVA) ที่มี vision encoder ดีกว่ามาก ตอบ วัตถุประสงค์ข้อ 3 (สร้าง caption ที่สะท้อน academic content)
  • Document-level split: เราจะยึดแนวทางของ SciCap+ ในการแบ่ง train/test แบบ document-level เพื่อความน่าเชื่อถือของผล
  • Ablation design: เราออกแบบ experiment ตาม SciCap+ — ทดสอบ context levels: (1) figure-only → (2) +OCR → (3) +mention → (4) +full RAG → พิสูจน์ว่าแต่ละระดับช่วยเพิ่มคุณภาพขนาดไหน ตอบ วัตถุประสงค์ข้อ 4

Survey: Multimodal Information Fusion for Chart Understanding

Survey 2025

🎯 Overview

Survey paper ล่าสุด (2025) ที่ให้ภาพรวมครบถ้วนของ Chart Understanding ด้วย MLLMs ครอบคลุม evolution, limitations, และ future directions ตั้งแต่ classic DL ถึง modern MLLM paradigms

📌 จุดประสงค์หลักของ Paper

ปัญหาที่ Paper นี้แก้
  • Field กระจัดกระจาย: งานวิจัยด้าน chart understanding มีหลาย sub-tasks (QA, captioning, data extraction) แต่ขาด unified taxonomy และ cross-task comparison
  • ขาดการ map ว่า MLLM แต่ละ architecture เหมาะกับ task ไหน: นักวิจัยใหม่ไม่รู้ว่าควรเลือก approach ใดสำหรับ task ที่ต้องการ
  • Limitations ของ current models ยังไม่ได้ถูก characterize อย่างเป็นระบบ: ทุกคนรู้ว่า model มีปัญหา hallucination แต่ไม่ได้วิเคราะห์ว่ามาจากอะไรกันแน่
  • เป้าหมายที่ 1: สร้าง comprehensive taxonomy ของ chart understanding tasks แบ่งเป็น Basic (Classification, Element Extraction, Factoid QA) และ Advanced (Captioning, Chart-to-Table, Open-ended QA)

🏗️ MLLM Architecture Evolution

  • Component-Assembled VLMs: Donut, Pix2Struct, Matcha, UniChart, ChartInstruct
  • Adapting General-Purpose MLLMs: Fine-tuning (LoRA/SFT) + Knowledge Distillation
  • Prompting Strategies: Chain of Chart Reasoning (CCR), CoT, Program of Thoughts (PoT)
  • Intermediate Representations: Data tables (DePlot), Python code (ChartCoder), SVG/XML
  • MoE (Mixture of Experts): ChartMoE — dynamic routing ตาม query type
  • Reinforcement Learning: DPO, GRPO — เพิ่ม reasoning + ลด hallucination

⚠️ Key Limitations of Current MLLMs

ปัญหาหลักที่พบ
  • Perceptual Deficits: Vision Encoder ไม่สามารถจับ fine-grained details (grid lines, exact values)
  • OCR Shortcut: Models พึ่ง text labels มากเกินไป แทนที่จะ reason จาก visual data
  • Hallucination: สร้างข้อมูลที่ดูสมเหตุสมผลแต่ไม่ตรงกับ chart จริง
  • Lack of Robustness: เปลี่ยน style เล็กน้อย (สี, font) ทำให้ performance ตกลงมาก
  • Weak Multi-hop Reasoning: ยังทำ causality inference, prediction ได้ไม่ดี

  • Discriminative Fine-Tuning with Hard Negatives
  • Fine-Grained Visual Grounding (region-level alignment)
  • RL for Chain-of-Thought reasoning
  • Multi-Agent Debate for verification
  • Dynamic Resolution + Synthetic Data Augmentation

Synthesis & Research Direction

GAP IDENTIFIED

Context is King

SciCap+ พิสูจน์ว่า mention-paragraphs สำคัญกว่า visual features (ResNet) แต่ยังใช้แค่ regex matching ง่ายๆ → RAG สามารถดึง context ที่เกี่ยวข้องได้ดีกว่ามาก และ modern VLM ทำ figure understanding ได้ดีกว่า ResNet อีกมาก

METHODOLOGY

Hybrid RAG + VLM

รวม DePlot (chart→table) + SciCap+ (document context) + MLBCAP (multi-stage pipeline): ใช้ VLM สำหรับ perception, RAG สำหรับ context retrieval, LLM สำหรับ generation — แต่ละ component ที่เราเลือกมีงานวิจัยรองรับ

INNOVATION

Open-Source Pipeline

MLBCAP พึ่ง proprietary LLMs → เราใช้ open-source VLM (InternVL, LLaVA) + Docling สำหรับ parsing → reproducible, cost-effective, customizable และเป็น contribution ในแง่ accessibility ของ research

EVALUATION

Multi-level Assessment

ใช้ทั้ง automatic metrics (BLEU, CIDEr, RMS จาก DePlot) + human evaluation + LLM-as-judge (จาก MLBCAP) ตาม survey recommendations — ครอบคลุม วัตถุประสงค์ข้อ 4 ของวิทยานิพนธ์อย่างสมบูรณ์

ABLATION

Context Level Study

ออกแบบ experiment ตาม SciCap+ ablation pattern: (1) Figure-only (2) +OCR (3) +Mention (4) +Full RAG context → พิสูจน์ hypothesis อย่างเป็นระบบ ตอบ วัตถุประสงค์ข้อ 2 และ 3

SCOPE

Beyond Graph Plots

Survey ชี้ว่า field ขยายไปสู่ non-canonical charts → งานเราเริ่มจาก graph plots (ตาม SciCap) แล้วขยายไปครอบคลุม diagrams, flowcharts, scientific visualizations ในขั้นถัดไป
